# SSH MountMate v0.1.4

This release promotes the SAI/HPC workflow work from the pre-release series and adds macOS login mount support.

## What's Changed

- Add GitHub Release update checks in Settings and `--check-update`.
- Add an SAI cluster preset with app-managed SSH config generation.
- Optionally copy selected private keys into `~/.ssh` and write the copied `IdentityFile`.
- Keep SAI preset and Manual source fields blank until the user fills them.
- Mark required connection fields and validate them before saving.
- Block duplicate saved configs when IP/Host, user, and port are identical.
- Validate app-managed SSH config entries before writing them.
- Allow custom mountpoint paths with validation before save and before mount.
- Add configurable batch mount/unmount concurrency.
- Improve macOS unmount fallback for busy mountpoints with `diskutil unmount force`.
- Add macOS login mount support through user LaunchAgents.
- Add macOS startup helper version checks, explicit setup errors, and copyable startup error details.
- Prefer Lustre project quota for capacity display when the mounted remote path is on Lustre and has a nonzero hard block limit.
- Fall back to `rclone about` when Lustre project hard block limit is unavailable.

## Contributors

- Thanks to @MoseyQAQ for contributing the macOS login mount work in PR #2.

## Notes

- macOS login mount takes effect on the next login.
- Lustre capacity display intentionally uses the mounted remote root's project quota only. Subdirectories with separate project quotas are not reflected in the drive-level capacity bar.

# SSH MountMate v0.1.4 中文说明

这个版本将 SAI/HPC 工作流相关改进从预发布版转为正式版，并新增 macOS 登录挂载支持。

## 主要更新

- Settings 中新增检查 GitHub Release 更新，并支持 `--check-update`。
- 新增 SAI 集群预设，支持生成应用托管的 SSH config。
- 可选把用户选择的私钥复制到 `~/.ssh`，并写入复制后的 `IdentityFile`。
- SAI 预设和手动来源不会继承之前 SSH config 的用户名等信息，改为等待用户填写。
- 标出必填连接字段，并在保存前校验。
- 如果 IP/主机、用户名和端口完全一致，会阻止保存重复配置。
- 写入应用托管 SSH config 前会先做更严格校验。
- 支持自定义挂载点路径，并在保存前和挂载前校验。
- 批量挂载/取消挂载并行数可配置。
- 改进 macOS 忙碌挂载点的取消挂载 fallback，增加 `diskutil unmount force`。
- 通过用户级 LaunchAgent 支持 macOS 登录挂载。
- macOS startup helper 增加版本检查、明确报错和可复制的启动项错误详情。
- 远端路径在 Lustre 上且 project 有非零 hard block limit 时，容量显示优先使用 Lustre project quota。
- Lustre project hard block limit 不可用时，回退使用 `rclone about`。

## 贡献者

- 感谢 @MoseyQAQ 在 PR #2 中贡献 macOS 登录挂载支持。

## 注意

- macOS 登录挂载会在下次登录时生效。
- Lustre 容量显示只使用挂载远端根目录的 project quota。子目录如果有独立 project quota，不会反映到整个盘符的容量条上。
