# SSH MountMate v0.1.5

This patch release improves Windows defaults, language detection, and dependency guidance after v0.1.4.

## What's Changed

- Fix automatic Chinese language detection on Windows by using system UI locale APIs in addition to Python locale values.
- Widen the Add/Edit config form labels so both English and Chinese labels fit better.
- Fix Add/Edit config dialog label clipping introduced in the v0.1.4 pre-release by avoiding fixed-pixel label boxes and wrapping source options into two columns.
- Change the SAI preset mountpoint default back to Auto, so Windows uses an automatic free drive letter by default.
- Improve `~/mnt/name` folder mount handling:
  - create the `~/mnt` parent directory when needed;
  - keep the final Windows folder mount target uncreated;
  - choose suffixes such as `name-2` for built-in user-folder mounts when the target already exists.
- Make Windows dependency guidance friendlier for restricted networks:
  - keep automatic winget installation behavior;
  - show direct WinFsp download guidance in manual help and README;
  - include WinFsp manual install guidance when winget installation fails.

## Notes

- Windows folder mount targets still must not already exist; SSH MountMate creates the parent folder only.
- rclone remains bundled in release builds. WinFsp is still required separately on Windows.

# SSH MountMate v0.1.5 中文说明

这是 `v0.1.4` 之后的补丁版本，主要改进 Windows 默认行为、语言识别和依赖安装提示。

## 主要更新

- 修复 Windows 中文系统自动语言识别，除 Python locale 外也会读取系统 UI 语言。
- 加宽新增/编辑配置页左侧标签区域，中英文标签更不容易被截断。
- 修复 `v0.1.4` 预发布版引入的新增/编辑配置页字段名截断问题，避免固定像素 label，并把来源选项改为两列排列。
- SAI 预设挂载点默认改回 Auto，Windows 下默认使用自动空闲盘符。
- 改进 `~/mnt/name` 文件夹挂载：
  - 需要时自动创建 `~/mnt` 父目录；
  - Windows 下仍不预创建最终挂载目标目录；
  - 内置“用户文件夹”挂载目标冲突时，会尝试 `name-2` 这类后缀。
- 改进 Windows 依赖说明，适配 winget/GitHub 网络不稳定的环境：
  - 自动安装仍会优先尝试 winget；
  - 手动说明和 README 中补充 WinFsp 直接下载入口；
  - winget 安装 WinFsp 失败时会附带手动安装说明。

## 注意

- Windows 文件夹挂载的最终目标目录仍要求不存在；程序只创建父目录。
- Release 构建仍内置 rclone；Windows 上仍需要单独安装 WinFsp。
