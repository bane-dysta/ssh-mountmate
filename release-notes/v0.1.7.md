# SSH MountMate v0.1.7

This release improves custom mountpoints and SSH config batch import controls.

## What's Changed

- Add an explicit custom mountpoint option in the Add/Edit config dialog.
- Keep Auto, `~/mnt/name`, drive-letter presets, and custom folder mountpoints in the same mountpoint control.
- Let the custom mountpoint browse button select a parent folder. SSH MountMate then fills a generated child mountpoint such as `parent/name`, adding suffixes when needed.
- Allow new SSH config batch import entries to be manually ignored instead of imported.
- Relax manual duplicate detection:
  - manual configs are only blocked as duplicates when IP/Host, user, port, remote path, and mountpoint are all the same;
  - the same SSH server can now be saved multiple times for different remote paths or local mountpoints.
- Keep SSH config batch import conservative:
  - an imported Host is treated as duplicate when it points to the same HostName/User/Port;
  - batch import does not ask for per-entry remote paths or local mountpoints, so these cases are left for manual configuration.

## Notes

- On Windows, custom folder mountpoints still require an existing parent directory and a target folder that does not already exist.
- On macOS/Linux, custom folders are created if missing, but they must be valid absolute paths or `~` paths.

# SSH MountMate v0.1.7 中文说明

这个版本改进了自定义挂载点和 SSH config 批量导入控制。

## 主要更新

- 新增/编辑配置页增加明确的自定义挂载点选项。
- Auto、`~/mnt/名称`、Windows 盘符预设和自定义文件夹现在都在同一个挂载点控件里。
- 自定义挂载点的浏览按钮改为选择父文件夹。SSH MountMate 会自动填入类似 `父文件夹/名称` 的子挂载点，必要时自动加后缀。
- SSH config 批量导入中的新条目现在也可以手动选择忽略，而不是只能导入。
- 放宽手动配置的重复判断：
  - 只有 IP/主机、用户名、端口、远程路径和挂载点都相同时，才会拦截为重复配置；
  - 同一台 SSH 服务器现在可以为不同远程路径或不同本地挂载点保存多份配置。
- SSH config 批量导入保持保守判断：
  - 只要导入的 Host 指向相同 HostName/User/Port，就认为是重复；
  - 批量导入不会逐项填写远程路径或本地挂载点，这类场景交给手动配置处理。

## 注意

- Windows 自定义文件夹挂载点仍要求父目录已存在，目标文件夹本身不能已存在。
- macOS/Linux 自定义文件夹不存在时会创建，但必须是合法的绝对路径或 `~` 路径。
