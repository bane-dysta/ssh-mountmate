# SSH MountMate v0.1.2

This release fixes macOS/Linux unmount behavior and improves macOS setup guidance.

## What's Changed

- Fix unmount on macOS and Linux. SSH MountMate no longer falls through to the Windows-only `taskkill` command.
- Use platform-native unmount commands:
  - macOS: `umount`, then `diskutil unmount`.
  - Linux: `fusermount3`, `fusermount`, then `umount`.
- Only terminate the recorded rclone PID after unmounting, reducing the risk of stopping unrelated rclone processes.
- Document macOS Gatekeeper/quarantine handling for downloaded builds.
- Document that Homebrew's rclone package cannot run `rclone mount` on macOS.
- Update the in-app manual rclone install help to recommend official rclone binaries on macOS instead of Homebrew.

## Assets

- `SSHMountMate-windows-x64.zip`
- `SSHMountMate-macos-arm64-x64.zip`
- `SSHMountMate-linux-x64.zip`

## macOS Notes

If macOS blocks the downloaded app, remove the quarantine attribute after unzipping:

```bash
sudo xattr -r -d com.apple.quarantine /path/to/SSHMountMate*
```

For rclone on macOS, use the official rclone binary:

```bash
curl https://rclone.org/install.sh | sudo bash
```

Do not use `brew install rclone` for SSH MountMate mounts; Homebrew rclone cannot run `rclone mount` on macOS.

# SSH MountMate v0.1.2 中文说明

这个版本修复 macOS/Linux 取消挂载问题，并改进 macOS 安装提示。

## 主要更新

- 修复 macOS 和 Linux 上取消挂载失败的问题。SSH MountMate 不会再误走到 Windows 专用的 `taskkill` 命令。
- 按平台使用对应的取消挂载命令：
  - macOS：`umount`，然后尝试 `diskutil unmount`。
  - Linux：`fusermount3`、`fusermount`，然后尝试 `umount`。
- 取消挂载后只清理记录中的 rclone PID，降低误杀其他 rclone 进程的风险。
- README 补充 macOS Gatekeeper/quarantine 处理方式。
- README 说明 Homebrew 版 rclone 在 macOS 上不能执行 `rclone mount`。
- 程序内的 rclone 手动安装提示改为 macOS 推荐官方 rclone 二进制，不再推荐 Homebrew。

macOS 上请使用 rclone 官方二进制：
