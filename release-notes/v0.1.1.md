# SSH MountMate v0.1.1

This release focuses on mount reliability, cross-platform mount paths, Windows installer encoding, and smaller build artifacts.

## What's Changed

- Run single mount and unmount actions asynchronously so the GUI stays responsive.
- Run batch mount and unmount operations concurrently.
- Prevent duplicate operations on the same config during batch or single actions.
- Add cross-process file locks for rclone config writes and per-config mount operations.
- Add a Windows `User folder (~/mnt/name)` mount option.
- Use `~/mnt/<name>` style mount paths on macOS and Linux.
- Keep saved config names unique to avoid `~/mnt/<name>` path conflicts.
- Improve Windows dependency installer encoding by forcing UTF-8 for cmd, PowerShell, and logs.
- Subset the bundled Chinese font during build, reducing package size significantly.

## Assets

- `SSHMountMate-windows-x64.zip`
- `SSHMountMate-macos-arm64-x64.zip`
- `SSHMountMate-linux-x64.zip`

## Notes

- rclone is still installed separately by the app helper instead of being bundled.
- Windows rclone mounts still require WinFsp.
- macOS and Linux mounts require the platform's FUSE support.

# SSH MountMate v0.1.1 中文说明

这个版本主要改进挂载可靠性、跨平台挂载路径、Windows 安装脚本编码，以及构建产物体积。

## 主要更新

- 单个挂载/取消挂载改为后台异步执行，避免 GUI 卡顿。
- 批量挂载/批量取消挂载改为并发执行。
- 防止同一个配置在批量或单点操作中被重复挂载。
- 为 rclone 配置写入和单配置挂载操作加入跨进程文件锁。
- Windows 新增 `User folder (~/mnt/name)` 挂载方式。
- macOS 和 Linux 使用 `~/mnt/<名称>` 风格的挂载路径。
- 配置名称保持唯一，避免 `~/mnt/<名称>` 路径冲突。
- Windows 依赖安装脚本强制使用 UTF-8，降低中文乱码概率。
- 构建时对子集化中文字体，显著减小安装包体积。

## 下载内容

- `SSHMountMate-windows-x64.zip`
- `SSHMountMate-macos-arm64-x64.zip`
- `SSHMountMate-linux-x64.zip`

## 注意

- rclone 仍由应用辅助安装，不内置在程序里。
- Windows 上 rclone mount 仍需要 WinFsp。
- macOS 和 Linux 上挂载需要系统具备对应 FUSE 支持。
