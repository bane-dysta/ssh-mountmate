# SSH MountMate v0.1.3

This release improves out-of-the-box dependency handling and splits release packages by CPU architecture.

## What's Changed

- Bundle the official rclone binary in release builds.
- Keep a managed rclone download fallback when the bundled binary is unavailable.
- Add third-party notices and rclone MIT license text.
- Split release artifacts by platform and CPU architecture:
  - `SSHMountMate-windows-x64.zip`
  - `SSHMountMate-windows-arm64.zip`
  - `SSHMountMate-macos-x64.zip`
  - `SSHMountMate-macos-arm64.zip`
  - `SSHMountMate-linux-x64.zip`
  - `SSHMountMate-linux-arm64.zip`
- Detect mount-layer dependencies across platforms:
  - Windows: WinFsp
  - macOS: macFUSE
  - Linux: FUSE / `/dev/fuse` / `fusermount`
- Detect Linux distribution families from `/etc/os-release`.
- Show the recommended Linux FUSE/OpenSSH install command first.
- Show install commands only for missing dependencies. For example, if rclone is bundled and only FUSE is missing, the GUI no longer shows rclone install commands.
- Document CPU architecture checks and distro-specific dependency commands in the README.
- Keep release zip packages to a single platform executable.
- Embed bundled third-party notices and the rclone MIT license in the app.
- Add `--licenses` and a Settings button for viewing bundled license text.

## Notes

- macOS still requires macFUSE for mounting.
- Windows still requires WinFsp for mounting.
- Linux still requires FUSE support from the system.
- rclone is bundled, but system mount layers cannot be embedded in the app.
- The rclone MIT license is retained and accessible in-app; it is not shipped as a separate file inside the release zip.

# SSH MountMate v0.1.3 中文说明

这个版本改进开箱即用的依赖处理，并按 CPU 架构拆分发布包。

## 主要更新

- Release 构建内置官方 rclone 二进制。
- 如果内置 rclone 不可用，仍保留托管下载 fallback。
- 增加第三方声明和 rclone MIT 许可证文本。
- 按平台和 CPU 架构拆分发布包：
  - `SSHMountMate-windows-x64.zip`
  - `SSHMountMate-windows-arm64.zip`
  - `SSHMountMate-macos-x64.zip`
  - `SSHMountMate-macos-arm64.zip`
  - `SSHMountMate-linux-x64.zip`
  - `SSHMountMate-linux-arm64.zip`
- 跨平台检测挂载层依赖：
  - Windows：WinFsp
  - macOS：macFUSE
  - Linux：FUSE / `/dev/fuse` / `fusermount`
- 通过 `/etc/os-release` 识别 Linux 发行版家族。
- Linux 上优先显示当前发行版对应的 FUSE/OpenSSH 安装命令。
- 只显示缺失依赖的安装命令。例如 rclone 已内置、只缺 FUSE 时，GUI 不再提示安装 rclone。
- README 补充 CPU 架构识别和各发行版依赖命令。
- Release zip 包内只保留对应平台的单个可执行文件。
- 第三方声明和 rclone MIT 许可证文本嵌入应用内。
- 新增 `--licenses` 和 Settings 页面查看许可证按钮。

## 注意

- macOS 挂载仍需要 macFUSE。
- Windows 挂载仍需要 WinFsp。
- Linux 挂载仍需要系统提供 FUSE 支持。
- rclone 可以内置，但系统挂载层无法内置到应用里。
- rclone MIT 许可证仍然保留并可在应用内查看；不再作为 release zip 内的单独文件分发。
